# Terminology Alignment — Round 4

## Objective

Replace all remaining "cycle" terminology in agent files with canonical domain terms, as required by the CONTEXT.md glossary introduced in Round 3.

## Changes Made

- **Scope**: 4 agent definition files — Orbit.agent.md, Orbit Backlog.agent.md, Orbit Next Advisor.agent.md, Orbit Round.agent.md.
- **Replacements**: 6 occurrences of "cycle" replaced.
- **Notable decision**: "auto-fix cycles" was replaced with "auto-fix attempts" to avoid domain term collision while preserving the intended meaning.

## Validation

- 85 tests (38 regression + 47 smoke), 0 failures.

## Lessons Learned

1. **Glossary-first approach pays off**: Establishing a domain glossary (CONTEXT.md) early prevents terminology drift across files. The glossary created in Round 3 made this cleanup straightforward.
2. **Common-English vs. domain-term collision**: When an avoided domain term ("cycle") has a legitimate common-English use ("auto-fix cycles"), review should note it as Info rather than Warning. This prevents false positives during review.

## Task Arc Context

This was the final round (Round 4) of a 4-round task arc that delivered: carry-over fixes, backlog system, auto-route skill, residual risk cleanup (including CONTEXT.md), and terminology alignment. Final state: 85 tests, 0 failures, no open risks.
