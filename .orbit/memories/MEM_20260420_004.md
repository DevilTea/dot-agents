# Deterministic Timestamps, Duplicate-Slug Guard, and CONTEXT.md Domain Glossary

## Context

Round 0003 of task 2026-04-20_21-26-31 addressed three residual risks identified after the backlog system implementation.

## Changes

### 1. Deterministic Test Timestamps

- Added optional `createdAt` parameter to `addBacklogItem` in `backlog.mjs`, allowing callers to inject a fixed timestamp.
- Updated `regression-test.mjs` to pass explicit timestamps, eliminating non-deterministic sort bugs caused by same-millisecond `Date.now()` collisions.

### 2. Duplicate-Slug Guard

- Imported `access` from `node:fs/promises` in `backlog.mjs`.
- Added an existence check before writing a new backlog item file; throws an error if the slug already exists.
- Added 2 new regression tests covering the duplicate-slug rejection path.

### 3. CONTEXT.md Domain Glossary

- Created `plugins/orbit/CONTEXT.md` with 15 Orbit-specific terms (Round, Phase, Task, Dispatcher, etc.).
- Establishes a single source of truth for domain language to prevent terminology drift across agents and skills.

## Files Changed

- `backlog.mjs` — access import, createdAt passthrough, duplicate guard
- `regression-test.mjs` — explicit timestamps, 2 new duplicate-slug tests
- `CONTEXT.md` — new glossary file

## Review Outcome

0 critical, 0 warning, 3 info findings. 85 tests (38 regression + 47 smoke), 0 failures.

## Residual

Agent descriptions use "cycle" inconsistently with glossary term "Round" (cosmetic only).

## Lessons Learned

- **Explicit timestamps eliminate non-deterministic sort bugs.** When test ordering depends on creation time, injecting fixed timestamps is more reliable than relying on `Date.now()`.
- **Domain glossaries should be created early.** Establishing shared vocabulary before agents and skills proliferate prevents terminology drift and reduces review friction.
