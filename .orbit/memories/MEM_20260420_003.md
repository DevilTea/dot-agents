# Auto-Route Skill Creation (Round 2)

## Changes Delivered

1. **State manager extension** — Added `abandoned` to `ALLOWED_STATUSES` in `state-manager.mjs`, enabling rounds to be marked as abandoned for recovery routing.
2. **orbit-auto-route/SKILL.md** — New skill with a 4-branch decision tree:
   - Interrupted Round Recovery (incomplete round detected)
   - Completed Round → Next Advisor (round completed, needs post-round processing)
   - Backlog Available (no active round, backlog has items)
   - Nothing to Do (clean state, prompt user)
3. **Dispatcher update** — Updated `Orbit.agent.md` with a Required Skills table and a new auto-route evaluation step in the Dispatch Procedure.
4. **Regression tests** — Added 7 regression tests for auto-route contracts.

## Fix Cycle

- **Critical ordering bug**: Auto-route evaluation must happen _before_ round creation in the Dispatch Procedure. Placing it after caused new rounds to be created before recovery could trigger.
- **3 warnings resolved**: Added `next` phase to the recovery table, added missing functional test, added ordering verification test.

## Final Validation

- Smoke: 47/47 passed
- Regression: 35/36 (1 pre-existing failure)

## Lessons Learned

- **Dispatch Procedure step ordering is critical** — inserting a new step in the wrong position can silently break existing flows. Always verify positional ordering with explicit tests.
- **Testing positional ordering** — Dedicated regex-based tests that verify step A appears before step B in the agent markdown prevent ordering regressions.
- **Splitting compound steps** — When inserting new steps between existing ones, it may be necessary to split a compound step into separate sequential steps to create the correct insertion point.
