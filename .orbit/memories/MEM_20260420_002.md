# Backlog System Implementation with Carry-Over Fixes

## Objectives Addressed

1. **Carry-over fixes** — README corrected from 5 to 4 phases; `ALLOWED_MODES` added with mode validation in `state-manager.mjs`.
2. **Backlog system** — Full implementation spanning paths, library, index, CLI commands, skill (`orbit-backlog-ops`), agent (`Orbit Backlog.agent.md`), dispatcher integration, and README documentation.
3. **Routing skill** (`orbit-auto-route`) — Deferred to a future round.

## Key Artifacts

- **backlog.mjs**: `listBacklog`, `addBacklogItem`, `getBacklogItem`, `removeBacklogItem`.
- **orbit-backlog-ops/SKILL.md**: Authoritative skill for backlog operations.
- **Orbit Backlog.agent.md**: Dedicated agent for backlog management.
- **regression-test.mjs**: 29 tests covering structural consistency.

## Design Decisions

- Backlog items use **markdown-with-frontmatter** format, consistent with the memory file format.
- Slug validation enforces **strict lowercase-kebab** regex.
- `addBacklogItem` **silently overwrites** on duplicate slug — documented design contract (not an accidental behavior).
- Regression tests are **independent from smoke tests**, providing complementary structural consistency checks.

## Validation

- Total: **76 tests, 0 failures** (smoke + regression combined).

## Residual Risks

- Duplicate slug overwrite behavior is untested.
- `--body-file` path traversal guard is untested.
- No `CONTEXT.md` established yet.
- Routing skill (`orbit-auto-route`) not yet implemented.

## Lessons Learned

- **body-file path traversal guard** pattern was reused successfully from the memory-archive implementation.
- Regression tests complement smoke tests by catching structural consistency issues that unit-style smoke tests miss.
